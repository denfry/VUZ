# Лабораторная работа 5

## Обработка событий в Java

### Цель работы

Познакомиться с моделью событий в графических компонентах и научиться обрабатывать возникающие события с помощью интерфейсов.

## Описание программы

Программа представляет собой расширенную форму "Нарушение ПДД" с реализацией обработки различных типов событий.

## Реализованные события

### 1. События фокуса (FocusEvent)

**Интерфейс:** `FocusListener`

**Обработка:**
- При получении фокуса элементом (метод `focusGained()`) - устанавливается светло-желтый фон компонента
- При потере фокуса элементом (метод `focusLost()`) - восстанавливается белый фон

**Зарегистрировано для:**
- Все текстовые поля (п. ПДД, ст. КОАП, Код нарушения)
- Текстовая область (Фактические сведения)

**Демонстрация:**
- Кликните на любое текстовое поле - оно подсветится желтым цветом
- Переключитесь на другое поле - предыдущее вернется к белому фону
- Все действия логируются в области "Лог событий"

### 2. События движения мыши (MouseMotionEvent)

**Интерфейс:** `MouseMotionListener`

**Обработка:**
- При перемещении мыши (метод `mouseMoved()`) - отображаются координаты курсора в метке
- При перетаскивании мыши (метод `mouseDragged()`) - координаты обновляются с пометкой о перетаскивании

**Зарегистрировано для:**
- Основная панель формы

**Демонстрация:**
- Перемещайте мышь по форме - координаты отображаются в реальном времени в метке "Координаты мыши"
- Перетаскивание мыши также отслеживается и логируется

### 3. События изменения текста (DocumentEvent)

**Интерфейс:** `DocumentListener`

**Обработка:**
- При вставке текста (метод `insertUpdate()`) - логируется изменение
- При удалении текста (метод `removeUpdate()`) - логируется изменение
- При изменении атрибутов (метод `changedUpdate()`) - логируется изменение

**Зарегистрировано для:**
- Все текстовые поля (п. ПДД, ст. КОАП, Код нарушения)
- Текстовая область (Фактические сведения)

**Демонстрация:**
- Введите текст в любое поле - каждое изменение будет зафиксировано в логе
- Удалите текст - это также будет отмечено в логе

### 4. Дополнительно: События окна (WindowEvent)

**Интерфейс:** `WindowListener` (используется `WindowAdapter`)

**Обработка:**
- `windowOpened()` - окно открыто
- `windowClosing()` - окно закрывается
- `windowClosed()` - окно закрыто
- `windowIconified()` - окно свернуто
- `windowDeiconified()` - окно восстановлено

**Демонстрация:**
- Все манипуляции с окном логируются в области событий

## Компоненты интерфейса

1. **Поля ввода** - п. ПДД, ст. КОАП, Код нарушения, Фактические сведения
2. **Метка координат мыши** - отображает текущие координаты курсора
3. **Область лога событий** - выводит все происходящие события с временными метками
4. **Кнопки** - Сохранить, Отмена, Заполнить тестовыми данными

## Тестирование событий

### Тест 1: События фокуса
1. Кликните на поле "п. ПДД (1)" - оно подсветится желтым
2. Кликните на другое поле - первое вернется к белому фону
3. Проверьте лог событий - должны быть записи о получении и потере фокуса

### Тест 2: События движения мыши
1. Перемещайте мышь по форме
2. Наблюдайте за изменением координат в метке
3. Проверьте лог - при перетаскивании мыши появляются записи

### Тест 3: События изменения текста
1. Введите текст в поле "Код нарушения"
2. Наблюдайте за логом - каждое изменение фиксируется
3. Удалите часть текста - это также будет зафиксировано

### Тест 4: События окна
1. Сверните окно - в логе появится запись
2. Восстановите окно - появится соответствующая запись
3. Попробуйте закрыть окно - увидите запись о закрытии

## Технические детали

### Использованные интерфейсы:
- `FocusListener` - для обработки событий фокуса
- `MouseMotionListener` - для обработки движения мыши
- `DocumentListener` - для обработки изменений текста (Swing)
- `WindowListener` / `WindowAdapter` - для обработки событий окна

### Паттерн проектирования:
- **Делегирование событий** - компоненты-источники не обрабатывают события сами, а делегируют это слушателям

### Особенности реализации:
- Для Swing компонентов используется `DocumentListener` вместо `TextListener` (который работает только с AWT)
- Используется `WindowAdapter` для упрощения обработки событий окна (не нужно реализовывать все методы)
- Все события логируются с временными метками для удобства отслеживания
