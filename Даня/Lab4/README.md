# Лабораторная работа 4

## Задание: Форма "Нарушение ПДД"

### Описание

Создано GUI приложение для ввода информации о нарушении правил дорожного движения с интерфейсом, соответствующим требованиям.

## Компоненты интерфейса

1. **п. ПДД** - два текстовых поля для ввода пункта правил дорожного движения с кнопкой "..." для выбора из списка
2. **ст. КОАП** - два текстовых поля для ввода статьи Кодекса об административных правонарушениях с кнопкой "..." для выбора из списка
3. **Код нарушения** - текстовое поле и выпадающий список (ComboBox) для выбора кода нарушения
4. **Фактические сведения** - многострочное текстовое поле (TextArea) для ввода подробной информации о нарушении
5. **Кнопка "Заполнить тестовыми данными"** - автоматически заполняет все поля тестовыми примерами
6. **Кнопка "Сохранить"** - сохраняет введенные данные и отображает их в диалоговом окне
7. **Кнопка "Отмена"** - очищает все поля после подтверждения

## Функциональность

- **Ввод данных вручную** - все поля можно заполнять вручную
- **Выбор из списка** - кнопки "..." открывают диалоги для выбора значений
- **Заполнение тестовыми данными** - кнопка автоматически заполняет все поля реалистичными тестовыми примерами
- **Валидация и сохранение** - кнопка "Сохранить" выводит все введенные данные
- **Очистка формы** - кнопка "Отмена" очищает все поля

## Запуск программы

### Компиляция:
```bash
javac TrafficViolationForm.java
```

### Запуск:
```bash
java TrafficViolationForm
```

## Использование

1. **Ручной ввод:**
   - Заполните поля п. ПДД, ст. КОАП, код нарушения
   - Введите фактические сведения в текстовое поле
   - Нажмите "Сохранить" для просмотра введенных данных

2. **Использование кнопок выбора:**
   - Нажмите кнопку "..." рядом с полями п. ПДД или ст. КОАП
   - Выберите значение из выпадающего списка
   - Значение автоматически заполнится в соответствующие поля

3. **Заполнение тестовыми данными:**
   - Нажмите кнопку "Заполнить тестовыми данными"
   - Все поля автоматически заполнятся реалистичными примерами
   - Можно использовать эти данные для тестирования или как шаблон

4. **Очистка формы:**
   - Нажмите кнопку "Отмена"
   - Подтвердите действие
   - Все поля будут очищены

## Технические детали

- Используется `JFrame` для создания главного окна
- Компоненты размещены с помощью `GridBagLayout` для точного позиционирования
- Обработка событий реализована через `ActionListener`
- Используются диалоговые окна (`JOptionPane`) для выбора значений и вывода информации
- Многострочное текстовое поле с прокруткой для удобного ввода больших объемов текста
- Поддержка системного Look and Feel для нативного вида интерфейса
