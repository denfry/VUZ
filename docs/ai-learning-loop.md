# AI Learning Loop

Purpose: convert repeated failures into permanent repository improvements.

## Inputs to Monitor

- Repeated bugs.
- Failed or risky releases.
- Recurring code smells.
- Performance regressions.
- Migration incidents.
- Security findings.
- Developer friction patterns.
- Repeated review comments.
- Test failures.
- Architecture drift.

## Mandatory Conversion Rule

Every repeated issue must produce:
1. A new or updated repository rule.
2. A workflow update.
3. A checklist update.
4. A risk pattern entry.
5. A prevention strategy with owner/date.

## Loop Cadence

- Per-task: log incidents and lessons in PR/change notes.
- Weekly: classify patterns by severity and recurrence.
- Monthly: update governance docs and workflows.
- Quarterly: revise architecture and roadmap priorities.

## Evidence Format

For each pattern keep:
- symptom,
- trigger context,
- root cause,
- affected modules,
- fix applied,
- prevention rule/check added,
- verification status.

## Success Metrics

- Fewer repeat incidents per month.
- Faster time-to-diagnosis.
- Higher checklist compliance.
- Reduced hotfix frequency.
