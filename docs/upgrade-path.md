# Upgrade Path

## Objective

Provide a safe progression path from current educational modules to a maintainable, reusable platform baseline.

## Path

1. Governance adoption
- Apply AGENTS/workflow/checklist requirements in every task.

2. Module normalization
- Standardize run/build/config conventions across active JavaFX/JDBC labs.

3. Shared utility extraction
- Extract validation, config, and DB helper logic once duplication threshold is met.

4. Reliability hardening
- Add tests and CI checks for critical module flows.

5. Security and operations maturity
- Reduce committed secret exposure.
- Add release and rollback rehearsal routine.

## Exit Criteria per Stage

- Stage complete only when docs, checks, and verification evidence are committed and reviewable.
