# Workflow: Optimize Scaling

## Discovery
- Define scaling bottleneck (team, build, runtime, data).

## Impact Graph
- Map hotspots, shared modules, and operational constraints.

## Risk Analysis
- Evaluate throughput vs complexity tradeoffs.

## Compatibility Verification
- Preserve current behavior while improving scale characteristics.

## Rollback Path
- Use staged rollout with measurable checkpoints.

## Docs Updates
- Update CTO roadmap and architecture evolution documents.

## Test Expansion
- Add performance/regression checks where practical.

## Telemetry Validation
- Capture before/after metrics and failure rates.

## Post-Change Governance Updates
- Formalize new scaling practices into engineering laws.
