# Workflow: Design Feature

## Discovery
- Clarify user goal, scope, and non-goals.
- Identify impacted modules and existing constraints.

## Impact Graph
- List touched components, data paths, docs, and scripts.

## Risk Analysis
- Classify risks (compatibility, data, security, release).

## Compatibility Verification
- Confirm behavior preserved for existing user flows.

## Rollback Path
- Define commit-level rollback and config/data rollback.

## Docs Updates
- Update relevant README and `docs/*` governance artifacts.

## Test Expansion
- Add/plan tests for new behavior and regression coverage.

## Telemetry Validation
- Ensure start/success/failure logging points exist.

## Post-Change Governance Updates
- If new risk pattern appears, update laws/checklists/learning loop.
