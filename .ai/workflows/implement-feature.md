# Workflow: Implement Feature

## Discovery
- Confirm final design and acceptance criteria.

## Impact Graph
- Map code paths, config files, docs, and data dependencies.

## Risk Analysis
- Identify regression, migration, and performance risks.

## Compatibility Verification
- Validate unchanged behavior for existing flows.

## Rollback Path
- Keep patch atomic and reversible.

## Docs Updates
- Update README/changelog/release notes for behavior changes.

## Test Expansion
- Add or execute tests for modified behavior.

## Telemetry Validation
- Confirm logging for critical transitions and failures.

## Post-Change Governance Updates
- Promote repeated implementation pain points into new rules.
