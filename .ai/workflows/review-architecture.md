# Workflow: Review Architecture

## Discovery
- Capture current architecture and intended target.

## Impact Graph
- Trace dependencies and coupling hotspots.

## Risk Analysis
- Evaluate drift, ownership ambiguity, and scaling blockers.

## Compatibility Verification
- Ensure proposed changes preserve runtime expectations.

## Rollback Path
- Stage architecture changes with reversible milestones.

## Docs Updates
- Update `docs/architecture-evolution.md` and roadmap artifacts.

## Test Expansion
- Add checks for boundary and contract behavior.

## Telemetry Validation
- Verify observability on critical module boundaries.

## Post-Change Governance Updates
- Add boundary rules and checklist gates for drift prevention.
