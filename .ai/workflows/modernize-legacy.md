# Workflow: Modernize Legacy

## Discovery
- Define legacy boundary and modernization goal.

## Impact Graph
- Map old/new interfaces and dependent modules.

## Risk Analysis
- Identify migration and behavior parity risks.

## Compatibility Verification
- Maintain old contracts until migration completion.

## Rollback Path
- Keep dual-path operation where needed for fallback.

## Docs Updates
- Update architecture, migration, and upgrade docs.

## Test Expansion
- Add parity tests between legacy and modernized paths.

## Telemetry Validation
- Compare error/performance metrics before and after.

## Post-Change Governance Updates
- Document deprecation rules and removal deadlines.
