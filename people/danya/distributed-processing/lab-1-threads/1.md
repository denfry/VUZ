*Задание 1. Организовать синхронное взаимодействие потоков по использованию общего ресурса. Каждый поток делает фиксированное число операций, если в условии задачи это не указано. Все необходимые параметры необходимо задать в конструкторах класса*

1\.1 На складе имеется остаток некоторого товара. Один поток увеличивает запас на случайное число. Второй поток уменьшает запас на случайное число. Если нет достаточного количества, то отбор не производится. Число операций фиксированное

1\.2 Имеется складской остаток. Один поток увеличивает запас на случайное число. Второй поток уменьшает запас на случайное число. Если нет достаточного количества, то отбирается весь остаток. Количество операций первого потока фиксированное

1\.3 В кассе начальный остаток равен 0. Один поток делает фиксированное количество взносов наличными (на заданную величину). Второй поток забирает некоторую сумму (заданную величину). При отсутствии необходимых средств выплата наличных не производится

1\.4 В кассе имеется начальный остаток средств. Один поток делает взносы наличными (на заданную величину). Второй поток забирает некоторую сумму (заданную величину). При отсутствии необходимых средств выплата производится из имеющейся наличности. Число операций фиксированное

1\.5 На банковском счете есть начальная сумма. Один поток увеличивает сумму на некоторый взнос. Второй поток производит списание некоторой суммы. Допускается отрицательный остаток. Количество операций фиксированное

1\.6 На банковском счете есть начальная сумма. Один поток фиксированное число раз увеличивает сумму на некоторый взнос. Второй поток производит списание некоторой суммы. Если нет запрашиваемых средств, списывается весь остаток.

1\.7 На банковском счете есть начальная сумма. Один поток увеличивает сумму на случ. величину. Второй поток производит списание заданной суммы. Страховой (минимальный) остаток средств на счете должен гарантироваться. Количество операций фиксировано

1\.8 Два потока формируют номер документа, увеличивая предыдущий номер на 1. Номера документов не должны прерываться. Начальный номер - 0. Число операций - фиксированное.

1\.9 В кассе начальный остаток равен 0. Один поток делает взнос наличными (случ.величина) . Второй поток забирает фиксированную сумму. При выплате должен гарантироваться страховой остаток наличности в кассе. Количество операций первого потока фиксировано

1\.10 Имеется пункт проката автомобилей с заданным парком а/м

. Один поток со случайной интенсивностью (через случайный интервал времени) берет автомобиль в прокат. Другой поток также со случайной интенсивностью сдает автомобиль. Число операций фиксировано

1\.11 В магазин поступила заданная партия товара. Два потока производят покупку (объем товара определяется случайным числом). Если при покупке не хватает товара, то забирается весь остаток. Процесс завершается при исчерпании товара (но не более заданного числа операций).

1\.12 Касса производит выплату зарплаты. 3 потока имитируют получение зарплаты. Размер определяется случайным образом. При отсутствии заданной суммы выплачивается остаток. Процесс заканчивается при исчерпании наличных средств.

1\.13 В библиотеке есть запас книг заданного наименования. 3 потока читателей берут книги (1 в руки). При исчерпании запаса процесс прекращается

1\.14 В магазин поступил смартфон новой модели. 2 потока покупателей делают покупки (случайное число, ноне более 2 в руки). При исчерпании партии процесс прекращается

1\.15 2 клиента пользуются одним счетом. Каждый клиент случайным образом делает покупку или зачисление средств. Сумма покупки и зачисления определяется случайным образом. Необходимо гарантировать остаток средств на счете. Число операций фиксировано

1\.16 На автозаправке реализуется процесс поступления топлива и отпуска его клиентам. 1 поток заданное число раз производит пополнение топлива (фиксированного размера). 3 потока с заданным интервалом производят заправку (объем топлива определяется случайным образом). Если топлива не достаточно, то клиент получает отказ в заправке и поток останавливается. Процесс продолжается при наличии потоков на заправку

1\.17 На заправке имеется запас бензина. 5 потоков производят заправку. Объем заправки определяется случайным образом. При нехватке топлива заливается остаток. Процесс заканчивается при исчерпании запаса.

1\.18 В библиотеке имеется редкая книга в 2-х экземплярах. 3 потока читателей берут книгу и через случайный интервал времени возвращают. Если книги в наличии нет, то поток переходит в состояние ожидания. При возврате книги поток уведомляет другие потоки. Число операций потоков фиксированное-четное (книга должна быть возвращена)

1\.19 На заправке имеется запас бензина. 5 потоков производят заправку. Объем заправки определяется случайным образом. При отсутствии нужного объема заправка не производится. Процесс заканчивается после фиксированного числа заправок.

1\.20 Банк имеет фиксированный объем средств для кредитования. 3 потока клиентов пытаются получить кредит (размер кредита определяется случайным образом).  При отсутствии запрашиваемого объема выдается остаток. Процесс заканчивается при исчерпании объема средств 

1\.21 Интернет-магазин продает телевизоры. Один поток выполняет закупку телевизоров 1 раз (фиксированный размер партии). Два других потока выполняют фиксированное число покупок (покупка только 1 экземпляра). При отсутствии телевизоров поток переходит в режим ожидания. Поток закупки извещает все потоки о поступлении товара. 

1\.22 Банк имеет фиксированный объем средств для кредитования. 3 потока клиентов пытаются получить кредит (размер кредита для каждого потока является определенным фиксированным числом, но не более остатка средств). Процесс заканчивается при выдаче в каждом потоке заданного числа кредитов

1\.23 Клиент имеет дебетовую карту с начальным остатком. Один поток производит начисление фиксированной суммой средств на карту. Второй поток делает покупку с оплатой по карте (стоимость покупки определяется случайным образом). Покупка может быть совершена только при наличии средств на карте. Число операций фиксировано

*Задание 2. Реализовать взаимодействие потоков по шаблону «производитель-потребитель». При выборе данных из файла предусмотреть символ конца файла*

2\.1 Производитель генерирует случайные целые числа. Потребитель подсчитывает количество четных чисел

2\.2 Производитель генерирует случайные целые числа. Потребитель подсчитывает количество отрицательных чисел

2\.3 Производитель генерирует случайные целые числа. Потребитель подсчитывает количество нечетных чисел

2\.4 Производитель генерирует случайные целые числа. Потребитель подсчитывает количество положительных чисел

2\.5 Производитель генерирует  случайные целые числа. Потребитель подсчитывает сумму чисел, больших А

2\.6 Производитель генерирует  случайные целые числа. Потребитель подсчитывает сумму чисел, кратных 3

2\.7 Производитель генерирует  случайные целые числа. Потребитель подсчитывает произведение ненулевых чисел

2\.8 Производитель считывает слова из файла. Потребитель определяет длину каждого слова.

2\.9 Производитель считывает слова из файла. Потребитель определяет максимальную длину слова.

2\.10 Производитель считывает слова из файла. Потребитель складывает из слов строку.

2\.11 Производитель считывает строки из файла.  Потребитель определяет количество слов в строке.

2\.12 Производитель случайным образом выбирает элементы из списка. Потребитель определяет частоту встречаемости заданного элемента.

2\.13 Производитель считывает числа из файла.  Потребитель вычисляет их сумму.

2\.14 Производитель случайным образом выбирает элементы списка. Потребитель записывает их в файл.

2\.15 Производитель случайным образом генерирует числа. Потребитель записывает их в файл.

2\.16 Производитель генерирует случайные целые числа от 1 до 10. Потребитель подсчитывает количество простых чисел

2\.17 Производитель считывает числа из файла. Потребитель выводит отрицательные элементы.

2\.18 Производитель случайным образом выбирает элементы из списка. Потребитель вычисляет сумму четных элементов.

2\.19 Производитель случайным образом выбирает элементы из списка. Потребитель вычисляет произведение положительных элементов.

2\.20 Производитель считывает слова из файла (могут встречаться знаки препинания). Потребитель складывает из слов предложения (после слов делается пробел).

2\.21 Производитель считывает числа из файла. Потребитель складывает числа и в конце результат выводит в консоль.

2\.22 Производитель считывает числа из файла. Потребитель вычисляет суммы положительных и отрицательных чисел и в конце результат выводит в консоль. 

2\.23 Производитель генерирует случайные целые числа. Потребитель подсчитывает среднее арифметическое этих чисел.
